      Guide & FAQ

      Praxis-Guide für JimmyBox Studio: Schnellstart, lokale Domains, CMS-Projekte, Datenbank-Workflows, Checkpoints, Upgrade-Hinweise und die wichtigsten Stellen für Fehleranalyse.

      Schnellstart Architektur Projekte Daten & DB Checkpoints Troubleshooting

      Schnellstart

      Ich arbeite alleine. Was muss ich tun?
      App öffnen, Start drücken, warten bis die VM läuft, unter Projekte ein Projekt anlegen und mit Öffnen starten. Alles läuft lokal in JimmyBox Studio.
      Wo liegen lokale Projekte?
      Standardmäßig in ~/JimmyboxStudio/Sites. Der Pfad kann in den Einstellungen geändert werden. Die alte JimmyBox/Vagrant-Installation wird nicht angefasst.
      Was brauche ich für den Alltag?
      Die Projekttabelle, den Datenbankbereich, Hosts und Checkpoints. Alle wichtigen Aktionen laufen lokal in der App.

      Architektur

      Was läuft lokal?
      Eine Lima-VM mit Ubuntu 22.04, Apache, PHP-FPM 7.4 bis 8.3, MariaDB, Redis und Composer. Lima ist in der App gebundelt.
      Wo liegen Projekte?
      Standard: ~/JimmyboxStudio/Sites/<domain>. In der VM ist das als /var/www/sites/<domain> gemountet.
      Welche Domains werden verwendet?
      Studio canonicalisiert führendes www.. Ein Projekt kunde.at bekommt automatisch Hosts, SSL-SAN und Apache-Alias für kunde.at und www.kunde.at.
      Was wird an /etc/hosts geändert?
      Nur der markierte JimmyBox-Block. Projekt-Hosts zeigen auf die echte VM-IP 192.168.64.x, nicht auf localhost.

      Projekte & CMS

      Was macht „Projekt erstellen"?
      Ordner, Docroot, vHost, SSL-Zertifikat, Datenbank, Setup-Descriptor und optional CMS-Scaffold werden angelegt. Wiederholen ist idempotent und repariert fehlende Runtime-Teile.
      Welche CMS werden unterstützt?
      WordPress, TYPO3, Contao, Drupal, Joomla und Plain PHP. Der Adapter bestimmt Docroot, Medien-Pfade, DB-Konfig und Setup-Routinen.
      Wie funktioniert „Öffnen"?
      Studio repariert vHost/SSL/Metadaten, aktiviert die Hosts-Einträge und öffnet die bevorzugte HTTPS-URL. Bei Apex-Domains wird https://www.domain geöffnet.
      Wie wechsle ich PHP?
      Die Dropdowns in der Projekttabelle schreiben den vHost auf den passenden PHP-FPM-Socket um und reloaden Apache. Zusätzliche Versionen kommen über Einstellungen → PHP-Version hinzufügen.

      Import, Datenbank & Medien

      Wie importiere ich eine Live-Seite?
      Code-Archiv und optional SQL-Dump wählen, Produktionsdomain eintragen. Die Domain bleibt unverändert; lokal zeigt sie per Hosts-Datei auf die VM. Das vermeidet spätere URL-Rewrites beim Deploy.
      Was macht der Import mit der CMS-Konfiguration?
      Die DB-Zugangsdaten werden auf die lokale MariaDB gesetzt. WordPress bekommt zusätzlich einen lokalen URL-Override, der den aktuellen Host aus dem Request verwendet.
      Wie komme ich in die Datenbank?
      In der App über Datenbank durchsuchen oder über den phpMyAdmin-Button. Login: Server localhost, Benutzer root, Passwort root.
      Wo liegen Medien?
      Im Setup-Descriptor unter mediaPaths[]. Checkpoints verwenden diese Pfade CMS-übergreifend, nicht nur WordPress-Uploads.

      Checkpoints

      Was speichert ein Checkpoint?
      Code-Referenz, DB-Dump, Medienarchiv und .jimmybox-studio/setup.json. Wiederherstellen legt vorher Sicherheitsbackups an.
      Wann sind Checkpoints sinnvoll?
      Vor riskanten Plugin-/CMS-Updates, großen Imports, Datenmigrationen oder anderen größeren Änderungen.
      Was bedeutet „anheften"?
      Gepinnte Checkpoints werden von der Aufräumlogik nicht gelöscht.

      Troubleshooting

      Domain zeigt die falsche Site
      Projekt einmal über Öffnen starten. Dadurch werden vHost, SSL-Aliase und Hosts-Einträge repariert.
      HTTPS-Warnung im Browser
      Lokale Zertifikate sind selbstsigniert. Die Ausnahme muss pro Domain einmal bestätigt werden.
      phpMyAdmin zeigt Konfigurationsspeicher-Warnung
      phpMyAdmin über den Button erneut öffnen. Studio legt Konfigurationsdatenbank, Tabellen und TempDir automatisch nach.
      DMG/App wird als beschädigt gemeldet
      Die unsignierte Test-DMG ist nicht notarisiert. Für Weitergabe braucht es Developer-ID-Signing und Notarisierung.

      Practical guide for JimmyBox Studio: quickstart, local domains, CMS projects, database workflows, checkpoints, upgrade notes, and the first places to check when something does not behave as expected.

      Quickstart Architecture Projects Data & DB Checkpoints Troubleshooting

      Quickstart

      I work alone. What do I need?
      Open the app, press Start, wait for the VM, create a project under Projects, then press Open. Everything runs locally in JimmyBox Studio.
      Where are local projects stored?
      Default: ~/JimmyboxStudio/Sites. You can change it in Settings. The legacy JimmyBox/Vagrant setup is not touched.
      What do I use day to day?
      The project table, database tools, hosts management, and checkpoints. All normal actions run locally in the app.

      Architecture

      What runs locally?
      A Lima VM with Ubuntu 22.04, Apache, PHP-FPM 7.4 through 8.3, MariaDB, Redis and Composer. Lima is bundled with the app.
      Where are projects stored?
      Default: ~/JimmyboxStudio/Sites/<domain>. Inside the VM this is mounted at /var/www/sites/<domain>.
      How are domains handled?
      Studio canonicalizes leading www.. A project client.com automatically gets hosts, SSL SANs and Apache aliases for both client.com and www.client.com.
      What changes in /etc/hosts?
      Only the marked JimmyBox block. Project hosts point at the real VM IP 192.168.64.x, not localhost.

      Projects & CMS

      What does Create Project do?
      It creates the folder, docroot, vHost, SSL certificate, database, setup descriptor and optional CMS scaffold. Running it again is idempotent and repairs missing runtime pieces.
      Which CMS are supported?
      WordPress, TYPO3, Contao, Drupal, Joomla and Plain PHP. The adapter defines docroot, media paths, DB config and setup routines.
      What does Open do?
      Studio repairs vHost/SSL/metadata, enables hosts entries and opens the preferred HTTPS URL. Apex domains open as https://www.domain.
      How do I switch PHP?
      The project table dropdown rewrites the vHost to the matching PHP-FPM socket and reloads Apache. Additional versions are installed from Settings → Add PHP version.

      Import, Database & Media

      How do I import a live site?
      Choose a code archive, optionally a SQL dump, and enter the production domain. The domain stays unchanged; locally it resolves to the VM through hosts. That keeps deploys free of URL rewrites.
      What does import change in CMS config?
      Database credentials are pointed at local MariaDB. WordPress also gets a local URL override that derives the URL from the current request host.
      How do I access the database?
      Use Browse & Query in the app, or the phpMyAdmin button. Login: server localhost, user root, password root.
      Where is media stored?
      In the setup descriptor under mediaPaths[]. Checkpoints use those CMS-aware paths, not only WordPress uploads.

      Checkpoints

      What does a checkpoint store?
      Code reference, DB dump, media archive and .jimmybox-studio/setup.json. Restore creates safety backups first.
      When should I create one?
      Before risky plugin/CMS updates, large imports, data migrations, or other larger changes.
      What does pin mean?
      Pinned checkpoints are not deleted by retention cleanup.

      Troubleshooting

      Domain shows the wrong site
      Open the project once via Open. That repairs vHost, SSL aliases and hosts entries.
      Browser shows an HTTPS warning
      Local certificates are self-signed. Confirm the exception once per domain.
      phpMyAdmin shows a configuration storage warning
      Open phpMyAdmin via the button again. Studio creates the config database, tables and TempDir automatically.
      DMG/App is reported as damaged
      The unsigned test DMG is not notarized. Distribution to other Macs needs Developer ID signing and notarization.
